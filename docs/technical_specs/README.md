# Technical Specs (as-built, AFTER coding)

After completing a major feature → 1 file `<feature>.md` for humans to read later:
- **What was built**: 2-3 sentence summary.
- **End-to-end flow**: request → ... → response, key files + functions.
- **Files changed**: path — role.
- **Decisions made during implementation**: things that came up in code (different from spec).
- **How to verify**: what to run to confirm.
- **Things to watch**: tech debt, TODOs, limitations.
